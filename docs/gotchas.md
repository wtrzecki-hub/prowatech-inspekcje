## Znane problemy / gotchas

- **R2 publiczny URL `pub-*.r2.dev` blokuje Python-urllib User-Agent** (utrwalone 2026-04-29 wieczór, Faza 17). Pierwszy run `extract_photos_2025.py` z hosta padł na **HTTP 403 Forbidden** dla wszystkich 175 download-ów PDF. Hipoteza początkowa „rate-limit r2.dev" odrzucona — `Invoke-WebRequest` z PowerShell zwracał 200 OK na ten sam URL. Walidacja w Pythonie: `urllib.request.urlopen(url)` → 403, `urllib.request.urlopen(Request(url, headers={'User-Agent': 'Mozilla/5.0'}))` → 200. Cloudflare R2 publiczny URL ma WAF/heurystykę blokującą domyślny `Python-urllib/3.13`. **Naprawa**: każde wywołanie `urlopen` opakować w `Request(url, headers={'User-Agent': 'Mozilla/5.0'})`. **Zasada**: dla bulk download z R2 publicznego URL z Pythona — zawsze ustawiaj browser-like UA, niezależnie od hosta (PowerShell/bash/sandbox).

- **`is_deleted` nulluje się domyślnie** — kolumna `is_deleted` w kilku tabelach ma default `NULL`, nie `FALSE`. Dlatego zapytania client-side filtrujące skasowane rekordy **muszą** używać:
  ```ts
  .not('is_deleted', 'is', true)
  ```
  a **nie**:
  ```ts
  .eq('is_deleted', false)   // ❌ gubi wszystkie rekordy z NULL
  ```
  To jest powtarzający się błąd — fix widać w commitach `adfb128` i `8dcac77`.

- **OAuth redirect na Vercel** — lokalny `localhost:3000` przy logowaniu Google OAuth zostaje przekierowany na URL Vercela. W praktyce: testuj feature'y związane z auth na deployu Vercel, nie lokalnie. Prawdopodobnie wynika z konfiguracji Redirect URL w Supabase → Auth → Providers → Google (wskazuje na produkcyjny URL).

- **Credentialsy Supabase hardkodowane** — po problemach z inliningiem `NEXT_PUBLIC_*` w buildzie Vercel, URL i anon key są twardo wpisane w `src/lib/supabase/client.ts` (commit `1a39485`). Przy rotacji klucza — pamiętać o update w kodzie, nie tylko w env.

- **Empty `SelectItem value=""`** — shadcn/ui Select nie toleruje pustych wartości. Używamy `'all'` jako sentinel dla "brak filtra" (commit `7fe2348`, `98824f4`).

- **Polskie znaki w PDF** — domyślny font jspdf nie ma polskich znaków. Projekt osadza Roboto przez font subset (patrz `src/fonts/`). Nowe endpointy PDF muszą wczytać i zarejestrować ten font.

- **DOCX API route wymaga runtime `nodejs`** — `export const runtime = 'nodejs'` na górze pliku. W Edge runtime nie działa `Buffer` ani biblioteka `docx`. Nauczone po 3 iteracjach (`6f955b0`, `3b58e34`, `329e6ca`).

- **SSR w Supabase** — `createClient()` **nie wolno** wywoływać w body komponentu ani na top-level module. Zawsze w `useEffect` albo przez Server Action / Route Handler. Commity `5d3aff5` i `e328fdb`.

- **Migracja `rated_power_kw` → `rated_power_mw`** — starsze dane lub seedy mogą mieć starą nazwę. Zmiana z commit `2719c85`.

- ~~**Bug formularza `/klienci/[id]`**~~ — **NAPRAWIONE 2026-04-26 commit `d7bb5d6`**. Przyczyna: prawdopodobnie kombinacja silent fail Supabase JS przy `.update()` bez `.select()` (PostgREST `Prefer: return=minimal` zwraca 204 No Content niezależnie od ilości zaktualizowanych wierszy) plus puste pola form przekazywane jako `""` zamiast `NULL`. - **Null bytes w plikach `src/` zacommitowane w HEAD** (wykryte 2026-04-24) — 80 plików w `src/` ma null bytes w wersji gitowej (HEAD). - **`tr -d '\0'` obcina ogony UTF-16 LE plików** (wykryte 2026-04-24) — klasyczny bug: plik UTF-16 LE kończący się pełnym znakiem ma bajty `X\0` lub `\0X` na końcu, które `tr -d '\0'` usunie częściowo — ostatnie bajty zostają odcięte, tracąc ogon. W sesji 2026-04-24 zepsuło mi to 13 plików (ogony typu `Separato`, `Skeleto`, `SheetDescrip` zamiast pełnych nazw). **Zasada**: do konwersji UTF-16 LE → UTF-8 używać `iconv -f UTF-16LE -t UTF-8`, nigdy `tr -d '\0'`.

Bug zjadany w `catch (e) { console.error(...) }`, `historical[]` zawsze `[]`, zakładka "Historia inspekcji" zawsze pokazywała empty state mimo dziesiątek wpisów w bazie. **Naprawa**: 4 zmiany w komponencie — interface `HistoricalRow` (kolumny), query select, filter `protocol_pdf_url IS NOT NULL` (zamiast nieistniejącego `is_deleted`), render `h.notes`/`h.protocol_pdf_url`. **Zasada**: po migracji DROP+CREATE tabeli z innym schematem, zawsze grep cały kod pod nowe vs stare nazwy kolumn (`grep -rn 'pdf_url\|summary_notes\|is_deleted' src/`). Catch'e w komponencie React mogą cicho zjadać błędy queryów RLS/schema, prowadząc do empty states bez sygnału.

- **Sandbox proxy r2.dev — pierwsze ~3 requesty mogą przejść, bulk pada na 403** (utrwalone 2026-04-29 popołudnie, Faza 17). Sandbox proxy `localhost:3128` blokuje `*.r2.dev` jak opisano w gotcha o R2 wcześniej, ALE: pojedyncze `curl` z bash sandboxa do publicznego URL R2 przechodzą (sample 3 PDFy). Bulk 367× `urllib.request.urlopen` z tego samego skryptu zwraca **HTTP 403 Forbidden** dla wszystkich requestów (testowane: 364 errors + 3 skipped jako z poprzedniego sample runa). Hipoteza: rate-limit po N połączeniach z proxy albo różne traktowanie nagłówków `User-Agent` curl vs Python. **Zasada**: dla bulkowych downloadów z R2 (≥10 plików) — uruchamiaj skrypt z hosta (PowerShell/cmd Windows-side), gdzie czysty internet bez proxy. Dla pojedynczego pobrania (test/sample) — sandbox OK.

- **PyMuPDF: raw `extract_image(xref)` nie obraca obrazu, użyj `page.get_pixmap(clip=bbox)`** (utrwalone 2026-04-29 popołudnie, Faza 17). Plik PDF może transformować obraz przez CTM (Coordinate Transformation Matrix) — np. surowe bytes JPEG to landscape 964x604, ale na stronie wyświetla się jako portret 196x313pt (PDF rotuje przy renderingu). `doc.extract_image(xref)` zwraca raw bytes BEZ transformacji — dostaniesz "obrócony" obraz. **Zasada**: gdy potrzebujesz obrazu w orientacji takiej jak na stronie (= jak widzi user), używaj `page.get_pixmap(matrix=fitz.Matrix(zoom, zoom), clip=bbox)`. Trade-off: re-rasteryzacja (lekka utrata jakości vs raw), ale "what you see is what you get". Dla 220 DPI bbox 196x313pt → ~600x955 px — w sam raz dla karty turbiny.
